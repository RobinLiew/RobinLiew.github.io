开发环境及开发工具

一、开发环境变量的配置

  • 不管是使用Eclipse ADT还是Android Studio进行开发，都需要配置环境变量。首先，我们了解一下需要配置的环境变量及其作用。

  • PATH环境变量：作用是指定命令（javac/java/javadoc等）搜索路径，在命令行下面执行命令如javac编译java程序时，它会到PATH变量所指定的路径中查找看是否能找到相应的命令程序。我们需要把jdk安装目录下的bin目录增加到 现有的PATH变量中，bin目录中包含经常要用到的可执行文件如javac/java/javadoc等待，设置好PATH变量后，就可以在任何目录下 执行javac/java等工具了。我们这里设定的PATH值为：

    Path 该系统变量的值设置为
%SystemRoot%/system32;%SystemRoot%;%SystemRoot%/System32/Wbem;%SYSTEMROOT%/System32/WindowsPowerShell/v1.0/;C:/Program Files/Common Files/Thunder Network/KanKan/Codecs;C:/Program Files/Microsoft SQL Server/90/Tools/binn/;C:/Program Files/Common Files/TTKN/Bin;C:/Program Files/Common Files/Teleca Shared;C:/Program Files/Java/jdk1.6.0_21/bin

    上述只C:/Program Files/Java/jdk1.6.0_21/bin是java的PATH变量，注意变量之间需要用”;”隔开。

  • JAVA_HOME环境变量。它指向jdk的安装目录，Eclipse/NetBeans/Tomcat等软件就是通过搜索JAVA_HOME变量来找到并使用安装好的jdk。这里JAVA_HOME为：

    JAVA_HOME
C:/Program Files/Java/jdk1.6.0_21

  • CLASSPATH环境变量。作用是指定类搜索路径，要使用已经编写好的类，前提当然是能够找到它们了，JVM就是通过CLASSPATH来寻找类的。我们需要把jdk安装目录下的lib子目录中的dt.jar和 tools.jar设置到CLASSPATH中，当然，当前目录“.”也必须加入到该变量中。这里CLASSPATH为：

    .;C:/Program Files/Java/jdk1.6.0_21/lib/dt.jar;C:/Program Files/Java/jdk1.6.0_21/lib/tools.jar

    CLASSPATH的设定，对JAVA的初学者而言是 一件棘手的事。所以Sun让JAVA2的JDK更聪明一些。你会发现，在你安装之后，即使完全没有设定CLASSPATH，你仍然能够编译基本的JAVA 程序，并且加以执行。所以一般可以不用设定该变量。

    可以在系统的dos窗口设置 set classpath=c:\myclass
该路径下有demo.class文件
此时运行java命令时，会优先寻找该临时环境变量下的类文件，如果没有，则在所在的目录下寻找

  • Java环境变量配置步骤：

    在windows桌面上右击“我的电脑” —> “属性” —> “高级” —> “环境变量”，在“系统变量”里我们可以看到系统的各个环境变量的值。双击某个变量名可以修改变量值，变量值之间用“;”隔开。我们还可以“新建”原先没 有的变量。与jdk有关的有3个环境变量；“JAVA_HOME”，“PATH”，“CLASSPATH”。我的系统中“PATH”变量已经存在，可以直 接添加新值（其它变量值不要动，防止其它程序运行异常），其它两个变量需要新建。

    CLASSPATH= .;%JAVA_HOME%/lib/dt.jar;%JAVA_HOME%/lib/tools.jar

    JAVA_HOME = C:/Program Files/Java/jdk1.5.0

    PATH = %JAVA_HOME%/bin;%JAVA_HOME%/jre/bin

    从上面我们可以看出，PATH和CLASSPATH都使用到了JAVA_HOME，也可以使用绝对路径，两者皆可。

  • 其他需要注意的小细节：

    • 对于AS这三个环境变量（由于Android Studio版本的升级，现在高版本已经不用配置CLASSPATH环境变量）必须都要配置，这要比ADT严格一些。
    • PATH，CLASSPATH等大小写无所谓，即大小写都可以。

    • 环境变量有系统变量和用户变量,设置成哪个都行,如果是系统变量的话对于所有登陆使用此电脑的。

    • 这一部分主要参考了http://blog.csdn.net/liranke/article/details/1954472 ，还有一些小细节也可以参考该博客。

二、开发工具-Android ADT

  • 首先，下载开发工具ADT和AS，可以在谷歌官网上进行下载。这里推荐一个网址http://www.androiddevtools.cn/ 或者http://www.android-studio.org/ 下载很方便。

  • Android开发环境的搭建：
    获取SDK工具包，包含的内容有：
    Eclipse+ADT插件（可被Android Studio代替）
    Android SDK
    Android Platform-tools
    最新的Android开发平台
    最新的模拟器镜像

    SDK工具包解压后包括三部分：Eclipse,sdk,SDK Manager.exe（或者为Android Studio，sdk两部分，SDK Manager.exe 在sdk中）。

    SDK Manager打开后，可以下载安卓模拟器（其中包含各种安卓版本，当然是用安卓手机调试也可以）等到sdk中。

    工具包中包括：
    工具：SDK工具集，SDK平台工具集，
    安卓模拟器:帮助文档，SDK平台，ARM架构的系统镜像，Intel架构的系统镜像，Android API的源码
    Extras：Android支持类库（为了低版本使用高版本的新特性），Intel模拟器加速器（解决模拟器运行速度慢的问题）

  • Eclipse和真机的连接：
    1.将手机中“设置”—“应用程序”——“开发选项”中的“USB调试”选中，不同的手机该选项名称和位置可能略有不同
    2.先将手机连接到电脑上，安装驱动，建议下载安装一个驱动精灵，它会自动检测手机驱动，根据提示点击安装即可。一般下个豌豆荚就可以自动安装驱动了。
    3.Eclipse显示出真机设备，可以开始调试

  • 模拟器和真机的网络调试：
    我们在本地PC机上搭建服务器，对模拟器而言，PC机的IP地址为：http://10.0.2.2:8080
    对整机而言，PC机的IP地址解释PC机在网络中的IP地址，（例如”http://192.168.1.104:8080）

    有时真机连接本机搭建的服务器时，会连接不上，可以尝试解决办法：关闭电脑安全卫士等软件，关闭windows系统防火墙。

  • Android Studio常用快捷键

  • 代码折叠（查找感兴趣的代码块）—— ctrl+N

  • 加标签 —— F11
  • 侧边栏Project切换 —— ctrl+1
  • 侧边栏Structure切换 —— ctrl+7
  • 侧边栏Favorites切换 ——ctrl+2
  • 循环显示已使用过的单词 —— Alt+/
  • 弹出generate窗口（包含构造器，set，get等方法）—— alt+insert
  • 补全代码 —— ctrl+shift+space
  • 查找 —— Ctrl + F
  • 查找+替换 —— Ctrl + R
  • 注释代码(//) —— Ctrl + /
  • 注释代码(/**/) —— Ctrl + Shift + /
  • 格式化代码 —— Ctrl + Alt + L
  • 快捷覆写方法 —— Ctrl + O
